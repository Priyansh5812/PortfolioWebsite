My Best Works

1. Seasonal Word Quest

The core gameplay revolves around generating optimized letter grids that guarantee multiple valid word paths, balancing randomness with deterministic constraints to ensure consistent challenge quality.This project demonstrates strong ownership across gameplay logic, backend integration, and live content pipelines—built with a production mindset rather than a prototype approach.

Word validation and search are implemented using efficient Trie and Hashmap-based structures, enabling fast lookups and smooth real-time gameplay even under tight time constraints. From a systems perspective, the project emphasizes backend-driven design. Firestore is used for real-time player data, leaderboard syncing, and progression tracking. Cloud Functions handle authoritative logic such as score verification, anti-cheat measures, and event orchestration. Authentication (Auth) ensures secure user identity and session handling.
The game is structured around continuous content delivery via live-ops. Seasonal events and dynamic rule variations are remotely configured, enabling rapid iteration without client updates. Monetization is integrated through In-App Purchases (IAP) and Ads, designed to coexist without disrupting core gameplay flow.
The architecture prioritizes modularity and scalability, allowing new game modes, events, and features to be layered without restructuring the core systems.

2. Custom ECS in C++

Designed and implemented a custom Entity-Component-System (ECS) architecture from scratch using C++ and raylib, with a strong emphasis on modularity, performance, and data-oriented design.
The system is built around a clear separation of concerns between Entities, Components, and Systems, enabling flexible composition of behavior.

Components are designed to be lightweight and data-driven, while systems operate on relevant component sets to define behavior, ensuring scalability and maintainability.
A structured lifecycle model—Start(), Update(), and OnDestroy()—was implemented across systems to standardize execution flow and simplify state management. This allows predictable behavior orchestration and easier debugging as the project scales.
The architecture follows Data-Oriented Design (DOD) principles, optimizing memory layout and iteration patterns for better cache efficiency and runtime performance. This makes it suitable for real-time applications such as games.
To validate the design, the ECS was integrated into a working prototype: a Space Invaders-style game, demonstrating entity management, system-driven behavior, and real-time updates in a practical environment.
This project reflects a strong understanding of low-level system design, runtime architecture, and performance-focused programming in C++.
